# Project Title

## CodeExplain: Plain-English Code Tutor

**Tech Stack**

* Python 3.12+
* Streamlit
* Any Gemini API
* Ollama (optional)
* LM Studio (optional)
* dotenv
* Requests/OpenAI SDK

---

# Project Goal

Create a web application where a student pastes code and receives:

* Plain-English explanation
* Line-by-line explanation
* Time Complexity
* Space Complexity
* Suggested improvements
* Translation into another language (English/Hindi for example)
* Quiz questions

Everything should appear in clearly separated sections.

The application should support multiple LLM providers without changing the application logic.

# Phase 4

## Prompt Engineering

This project heavily depends on one high-quality prompt.

The LLM should always produce identical formatting.

Instead of

```
Explain this code.
```

Use a structured prompt.

Prompt contains:

---

Role

```
You are an expert programming tutor.
Explain code for beginners.
```

---

Rules

```
Never skip sections.

Never merge sections.

Always estimate complexity.

Use markdown.

Keep explanations beginner friendly.
```

---

Required Output

```
1. Summary

2. Plain English Explanation

3. Line-by-line Explanation

4. Time Complexity

5. Space Complexity

6. Suggested Improvements

7. Translation

8. Quiz
```

---

This ensures consistency.

# Scope Boundaries (Keep It Student-Friendly)

To keep the project focused and achievable, **do not include**:

* User authentication or login
* Database storage
* Conversation/chat history
* File upload support (only paste code)
* Code execution or sandboxing
* Syntax highlighting customization
* RAG/vector databases
* Agent frameworks (LangChain, CrewAI, etc.)
* AI-generated code rewriting or auto-fixing
* Performance benchmarking
* Analytics dashboards

The application should do **one thing well**: accept a code snippet, send it to the selected LLM provider, and present the explanation in a clean, consistent, beginner-friendly format.
